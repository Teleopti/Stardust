{{vm.masterTenant.TenantName}} ETL initial configuration
Before it is possible to use the ETL manual tool or background service for processing reporting data a few important decisions must be made. Please select carefully as these, once set, cannot be undone easily without major maintenance work.

- Interval length: Highest reporting resolution, typically 15 minutes. This absolutely must match the interval used in the aggregation database, which in it self will most often mirror the interval used in the ACD reports (If interval length is already in use in Analytics database this option is disabled).

- Culture: The culture used for some report data, i.e, will decide day names and week numbering for all reporting users. Will not affect the date and numbering formats in reports.

- Time Zone: Should normally reflect the time zone of the server where ETL is running. Used by ETL to know exactly which data periods jobs should be run with.
