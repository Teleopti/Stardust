When the server installation is complete, navigate to the place on disc where the Admin client files are stored, normally C:\Program Files (x86)\Teleopti\TeleoptiCCC\Client

Open the file Teleopti.Ccc.SmartClientPortal.Shell.exe.config.deploy, that´s the config file, just with a deploy extension. Look for the following entries,

    <client> <!--

            To add more than one SDK, you must enter them here and give them different name, e. g.

      <endpoint address="http://SomeServer/TeleoptiCCC/SDK/TeleoptiCCCSdkService.svc"

      binding="basicHttpBinding"

                  contract="Teleopti.Ccc.Sdk.Common.Contracts.ITeleoptiCccSdkInternal"

                  bindingConfiguration="BasicHttpBinding_WithSession"

                                                    behaviorConfiguration="EnableAuthentication"

                                                    name="Some Custom Name">

      </endpoint>

             

      <endpoint address="http://AnotherServer/TeleoptiCCC/SDK/TeleoptiCCCSdkService.svc"

      binding="basicHttpBinding"

                  contract="Teleopti.Ccc.Sdk.Common.Contracts.ITeleoptiCccSdkInternal"

                  bindingConfiguration="BasicHttpBinding_WithSession"

                                                    behaviorConfiguration="EnableAuthentication"

                                                    name="Another Custom Name">

      </endpoint>

            -->

      <endpoint address="http://MainServerHostName/TeleoptiCCC/SDK/TeleoptiCCCSdkService.svc"

      binding="basicHttpBinding"

                  contract="Teleopti.Ccc.Sdk.Common.Contracts.ITeleoptiCccSdkInternal"

                  bindingConfiguration="BasicHttpBinding_WithSession"

                                                    behaviorConfiguration="EnableAuthentication"

                                                    name="Default Installation">

      </endpoint>

There you see the URL to the SDK entered during installation. Now you have the possibility to uncomment any of the lines above to enter one or more other SDK URLs. Just uncomment and enter the appropriate address. Don´t forget to use https if that´s needed.

Now you have modified the signed ClickOnce file and they are invalid so we need to sign them again. We will do that using a tool called ClickOnceSign.exe. You find that application under C:\Program Files (x86)\Teleopti\ClickOnce. Start the application.

First press the Browse button and point out the path where the client files are stored. That´s the same folder where we modified the config file, typically, C:\Program Files (x86)\Teleopti\TeleoptiCCC\Client. When you have done that the application will fill in the 2 text boxes below. Leave them like that. In the field Provider URL you enter the URL to where the client files will be downloaded, typically http://SERVERNAME/TeleoptiCCC/Client. The last step is to point out where the certificate is. We deploy a temporary certificate, TemporaryKey.pfx, found in folder C:\Program Files (x86)\Teleopti\ClickOnce. Just point to that and leave the Password field empty. If the customer has its own certificate, point to that and apply the appropriate password.

Press the Sign button and let it do it´s trick. You will be noted when it is finished. It takes approximately 10 seconds. You are done!

Uninstall the client, if you had it installed, and then reinstall it again.
